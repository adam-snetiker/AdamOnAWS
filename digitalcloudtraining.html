Lab 1: Identity & Access Management (IAM)

  • Created a new IAM user called adamdtc
  • Created a password so I can log in to this account through the AWS Management Console.
  • Generated an access key ID and secret access key for this account.
  • Added new user to a group called Training so I can grant access to the minimal number of resources necessary for completing labs as I need them.
  • To start with, I'm attaching two policies to this group:
    • IAMSelfManageServiceSpecificCredentials
    • IAMUserChangePassword
  • Added the following tags to identify this account so I know it is to be used for training labs from a specific course.
    • Key: Provider, Value: Digital Cloud Training
    • Key: Instructor, Value: Neal Davis
    • Key: Course, Value: AWS Certified Solutions Architect Associate
    • Key: Platform, Value: Udemy
  • This user will sign in through the custom link created under my room account: https://ajsnetiker.signin.aws.amazon.com/console
Problem
After looking closely, I realized that I accidentally set my user name to adamdtc instead of adamdct (the c and the t are in the wrong order). It should be DCT, as in "Digital Cloud Training".

Solution
I solved this problem by taking the following steps:
  1. Ran aws configure through the command line to update access keys to those of my new training account.
  2. Attached the policy IAMFullAccess to the Training group, which includes iam:UpdateUser, a policy that allows me to change the user name of this account programatically.
  3. Ran aws iam update-user --user-name adamdtc --new-user-name adamdct to execute the name changee.
  4. Detached the policy IAMFullAccess from the Training group to make sure full access was only temporary.
