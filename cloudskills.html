Project 2: Reusable and Clean Python Code

In this project, I used Python to create an S3 bucket with the Boto3 library, which is used to access AWS resources. First, I attempted to install boto3 using the command pip install boto3 in the terminal tab of VS Code. When running my code, I got an error message saying the module didn't exist, so I had to research a different way to solve the problem.

I found an alternative and set up the AWS SDK for Python using the following commands:

$ pip install virtualenv
$ cd boto3
$ virtualenv venv
...
$ . venv/bin/activate
$ python -m pip install -r requirements.txt
$ python -m pip install -e .

This solved the first problem, but when I ran the code again I got an error saying the unspecified location constraint is not compatible with the endpoint to which the command was sent. After researching this problem, I solved it by adding the following to my code:

CreateBucketConfiguration = {
   'LocationConstraint': 'us-west-1'
},

After solving this problem as well, I was met with an access denied error. To fix this, I went back to the console and signed in under my root account. I then granted the S3FullAccess policy to the IAM account I'm using to connect to AWS from my local machine.

A quick look at the S3 bucket list confirms successful creation of a new bucket called snetcloudnative. I can now reuse this code any time I want to create a new s3 bucket by running the command
python3 ./s3bucket.py bucketname

Project 3: Reusable and Clean PowerShell Code

For this project, the task is to write a PowerShell script that creates a new Azure resource group. Since I'm starting from scratch, I need to install both PowerShell and the Azure CLI.

In order to run Homebrew, I first had to convert the shallow clone to a full clone using the following commands:

git -C /usr/local/Homebrew/Library/Taps/homebrew/homebrew-core fetch --unshallow
git -C /usr/local/Homebrew/Library/Taps/homebrew/homebrew-cask fetch --unshallow

Before I install the Azure CLI, it appears I have to install Xcode Developer tools using the command xcode-select --install. After that, and once I have a full clone of Homebrew, I'm able to I'm able to successfully install the Azure CLI using brew update && brew install azure-cli.

To begin, I installed PowerShell using the command brew install --cask powershell and verified that it's working with pwsh. This is also used to start the shell, which is necessary because otherwise I'd be running commands in Bash. In VS Code, it appears the default shell is zsh.

When running my code, I ran into an error that said the module was not part of a cmdlet, script or file. Basically, PowerShell couldn't find the specific Azure CLI command I want to use, which is New-AzResourceGroup. To troubleshoot the problem, here are the steps I took:

  • Looked up how to install a PowerShell module
  • Found the name of the correct module that contains the New-AzResourceGroup command
  • Ran a search for the module and installed it once it was found. The correct module for this project is Az.Resources. The command I used to find and install the module is Find-Module -Name Az.Resources | Install-Module.
  • Added Import-Module Az.Resources to the top of my code to import the module.

In order to successfully run the code, I had to run Connect-AzAccount, which opened a browser tab so I could log in to Azure. Finally, after the connection was established, I ran New-ResourceGroup -rgName snetRG01 -location westus to create a resource group called snetRG01 in the West US region.

Last but not least, I was able to confirm successful creation of the resource group in the Azure console.

Project 4: Linting in PowerShell and Python

This project is all about linting, which means optimizing and cleaning up code so it follows best practices.

In Python, we start with pip install pylint to install the linter. To begin cleaning up code, I ran pylint ./s3bucket.py in the directory where my code is, and got my first round of results.

Each time pylint is run, it provides your code with a rating. My initial rating was 3.08/10, but after making corrections I ended up wit ha rating of 9.00/10. Some of the changes I made are:

  • Added an empty line at the bottom of the code
  • Placed import sys before import boto3
  • Added document strings to both the module and functions to describe what they do. For example, my module docstring is """This script creates an S3 bucket using the Boto3 library""".
  • Removed the bucket variable from the s3_bucket.create_bucket function because it was never used.

For PowerShell, I created a file called linting.ps1 that will be run against my code in New-ResourceGroup.ps1. I then took the following steps:
  • In the PowerShell Integrated Console, I ran Install-Module -Name PSScriptAnalyzer -RequiredVersion 1.18.3 to install it.
  • Imported the script analyzer module into the code using Import-Module PSScriptAnalyzer. I'm on a Mac, but was told that it's automatically included in Windows 10.
  • The entire linting script consists of one line of code: Imported it into the code using Invoke-ScriptAnalyizer -Path .. This will analyze all scripts in the current directory.

Project 5: Testing in PowerShell and Python

While linting is one form of testing the cleanliness of code, unit testing is another form to make sure it works as expected. In this project, we ran unit tests on the Python script to create a new S3 bucket and the PowerShell script that creates a new resource group.

Starting with PowerShell, the steps I took are as follows:
  • Created a new PowerShell script file called New-ResourceGroup_test.ps1
  • Installed Pester, a popular framework for various types of application testing, by running Install-Module -Name Pester -RequiredVersion 4.7.3
  • Run the test by using Invoke-Pester .\New-ResourceGroup_test.ps1 or by selecting the entire test function and selecting 'Run Selection' from the menu.
  • Fixed code after observing failed test.

The python tests go a little deeper and test the bucket name and region to make sure they both match the expected values and they're strings. To do this, I did the following:

  • Import the python unittest package
  • Select code and choose 'Run Python File in Terminal' from the menu to perform the test
  • Observe results, which state that 4 tests were run and the status is OK.

Project 6: Documenting Code

Documenting code is important for develoeprs to do so others can understand it. When using GitHub especially, the README file gives visitors an idea of what's in the repository, how to contribute, and provides other information. It's written in markdown format, which consists of several different things. # is used for heading, similar to H1 - H6 tags in HTML. In fact, the number of pound signs correspond to the heading level, so:
  • # = H1
  • ## = H2
  • ### = H3
  • #### = H4
  • ##### = H5
  • ###### = H6

As far as formatting goes, a * on either side of a string is used for bold text and _ for italics.

GitHub has an excellent guide called Mastering Markdown that contains the above syntax rules and more.


Week 3: Infrastructure as Code

Project 1: Building an ARM Template for Virtual Machine Deployment in Azure

ARM stands for Azure Resource Manager. Templates are used to create resources via code instead of manually clicking in the Azure interface. This is the process I followed to build my first one:

  • Install the extension called Azure Resource Manager (ARM) Tools in Visual Studio Code. This extension allows you to create template snippets on the fly that will be deployed via the Azure CLI.
  • Created a new file called template.json. Thanks to the extension, VS code automatically recognizes it as a JSON file. To start, we begin typing 'arm' and select the first item, 'arm!' from the menu to create a generic template, which contains the following sections:
    • $schema
    • parameters
    • functions
    • variables
    • resources
    • outputs
  • In the resources section, I began typing 'arm' again and selected 'arm-vm-ubuntu' from the list to automatically generate an Ubuntu Linux virtual machine. I then changed the default name, ubuntuVM1, to ajsnetVM1 to make it globally unique.
  • In the parameters section, typing 'arm' brings up an option to create a new-parameter, which generates a snippet containing a type and metadata such as a description. I named the paramter 'adminPassword' and set the descripton to 'Administrator Password'. Further down in the resources section under osProfile, I then set adminPassword to "[parameters('adminPassword')]". The list of parameters is available thanks to the ARM extension for VS code.
  • In the output section, I want to output the public IP and Fully Qualified Domain Name of the new virtual machine using "[reference(resourceId('Microsoft.Network/publicIPAddresses', 'ajsnetVM1-PublicIP')).dnsSettings.fqdn]"
  • Once complete, I entered a powershell terminal and logged into Azure using az login.

    • Note: If you have trouble logging in, you may have to specify your tenant ID, which can be found by searching 'tenant properties' in the Azure portal. Copy the Tenant ID, then run az login --tenant tenantID.

  • Once logged in, I ran az deployment group create --resource-group snetRG01 --template-file ./template.json -p adminPassword="********"
  • Finally, I can verify successful creation of the VM by visiting the Virtual machines service in my Azure console. The VM, along with all of the other resources that were provisioned, are listed in the snetRG01resource group.
