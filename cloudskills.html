Project 2: Reusable and Clean Python Code

In this project, I used Python to create an S3 bucket with the Boto3 library, which is used to access AWS resources. First, I attempted to install boto3 using the command pip install boto3 in the terminal tab of VS Code. When running my code, I got an error message saying the module didn't exist, so I had to research a different way to solve the problem.

I found an alternative and set up the AWS SDK for Python using the following commands:

$ pip install virtualenv
$ cd boto3
$ virtualenv venv
...
$ . venv/bin/activate
$ python -m pip install -r requirements.txt
$ python -m pip install -e .

This solved the first problem, but when I ran the code again I got an error saying the unspecified location constraint is not compatible with the endpoint to which the command was sent. After researching this problem, I solved it by adding the following to my code:

CreateBucketConfiguration = {
   'LocationConstraint': 'us-west-1'
},

After solving this problem as well, I was met with an access denied error. To fix this, I went back to the console and signed in under my root account. I then granted the S3FullAccess policy to the IAM account I'm using to connect to AWS from my local machine.

A quick look at the S3 bucket list confirms successful creation of a new bucket called snetcloudnative. I can now reuse this code any time I want to create a new s3 bucket by running the command
python3 ./s3bucket.py bucketname

Project 3: Reusable and Clean PowerShell Code

For this project, the task is to write a PowerShell script that creates a new Azure resource group. Since I'm starting from scratch, I need to install both PowerShell and the Azure CLI.

In order to run Homebrew, I first had to convert the shallow clone to a full clone using the following commands:

git -C /usr/local/Homebrew/Library/Taps/homebrew/homebrew-core fetch --unshallow
git -C /usr/local/Homebrew/Library/Taps/homebrew/homebrew-cask fetch --unshallow

Before I install the Azure CLI, it appears I have to install Xcode Developer tools using the command xcode-select --install. After that, and once I have a full clone of Homebrew, I'm able to I'm able to successfully install the Azure CLI using brew update && brew install azure-cli.

To begin, I installed PowerShell using the command brew install --cask powershell and verified that it's working with pwsh. This is also used to start the shell, which is necessary because otherwise I'd be running commands in Bash. In VS Code, it appears the default shell is zsh.

When running my code, I ran into an error that said the module was not part of a cmdlet, script or file. Basically, PowerShell couldn't find the specific Azure CLI command I want to use, which is New-AzResourceGroup. To troubleshoot the problem, here are the steps I took:

  • Looked up how to install a PowerShell module
  • Found the name of the correct module that contains the New-AzResourceGroup command
  • Ran a search for the module and installed it once it was found. The correct module for this project is Az.Resources. The command I used to find and install the module is Find-Module -Name Az.Resources | Install-Module.
  • Added Import-Module Az.Resources to the top of my code to import the module.

In order to successfully run the code, I had to run Connect-AzAccount, which opened a browser tab so I could log in to Azure. Finally, after the connection was established, I ran New-ResourceGroup -rgName snetRG01 -location westus to create a resource group called snetRG01 in the West US region.

Last but not least, I was able to confirm successful creation of the resource group in the Azure console.
